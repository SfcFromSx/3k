# 游戏设计文档：三国猜猜看 (Who Am I?)

## 1. 概述
- **游戏名称**: 三国猜猜看 - AI对决版
- **游戏类型**: AI模拟、推理、解谜、观战
- **平台支持**: Web (响应式设计)
- **核心概念**: 这是一个基于三国时代丰富历史和人物背景的自动化推理游戏。游戏完全由AI代理（Agents）进行对抗。三个AI玩家中的每一位都会被“裁判”（Judge）秘密分配一个**独特**的三国人物身份。玩家并不知道自己的身份，但能看到其他两位玩家的身份。在回合制机制下，每位玩家向“裁判”提出关于自己身份的“是/否”问题，并试图成为第一个正确猜出自己身份的玩家。人类用户则纯粹作为观众观战。

## 2. 角色与实体
- **裁判 (AI)**: 从包含200个人的三国数据库中，为每个AI玩家分配一个独特的秘密角色。裁判知晓所有三个身份，并回答每个玩家关于*该玩家自己*角色的问题。为防止泄露额外信息，裁判被严格限制只能使用三个简单的短语来回答：“是”（Yes）、“否”（No）或“不是，但也算不上错”（Not Yes and Not Wrong）。由人类专门选择的一个大语言模型（LLM）驱动。
- **AI玩家A, B, C**: 三个独立的推理代理，每一个都由本地Ollama实例中独立选择的LLM驱动。每个玩家被分配一个他们必须推理出的秘密身份。他们能看到其他两位玩家的身份，唯独看不到自己的。他们轮流向裁判询问关于自己的问题，并必须运用逻辑和排除法来找出自己是谁。**至关重要的是，随着时间的推移，这些代理会进行自我反思：它们会审查历史比赛的聊天日志，以更新一个持久的“技能文件”（skill file），从而不断发展和迭代它们的推理策略。**
- **翻译器 (AI)**: 一个专门负责实时翻译聊天日志的LLM。启用翻译后，聊天竞技场中的每条英文消息后面都会附带其中文翻译。
- **观众 (人类)**: 观看比赛，在AI代理协商时阅读聊天记录。在比赛开始前配置游戏参数、LLM分配以及翻译设置。

## 3. 可配置参数
在游戏会话开始前，人类观众可以配置以下内容：

### 游戏规则
- **回合数**: 一次会话中玩多少局（例如：3、5、10局）。每一局都会分配一组全新的3个角色。
- **每局提问次数上限**: 每个玩家在每局中拥有的最大提问回合数（例如：5、10、15次）。一旦玩家耗尽提问次数，他们必须进行最后一次盲猜或弃权放弃。

### LLM分配 (通过本地Ollama)
系统从本地Ollama实例中获取可用模型列表，并将其作为下拉选项供用户选择：
- **玩家A LLM**: 选择驱动玩家A的Ollama模型。
- **玩家B LLM**: 选择驱动玩家B的Ollama模型。
- **玩家C LLM**: 选择驱动玩家C的Ollama模型。
- **裁判 LLM**: 选择驱动裁判的Ollama模型。
- **翻译器 LLM**: 选择驱动翻译器的Ollama模型。

### 语言与翻译
- **聊天翻译**: 打开/关闭开关。启用后，翻译器LLM会将每条聊天消息实时翻译成中文，并在原始英文消息下方以较浅颜色的斜体字内联显示。

## 4. 核心游戏循环
1. **游戏设置 (人类输入)**: 人类观众配置局数、每局提问次数上限，以及LLM模型选择和翻译开关。
2. **回合初始化**: 裁判秘密且不重复地为三个AI玩家各分配一个角色。这些分配会展示给观众和其他玩家（但对被分配的玩家本人隐藏）。
3. **回合制调查**: 以轮流方式（A -> B -> C），当前回合的玩家向裁判提出一个关于自己秘密身份的“是/否”问题。
4. **裁判判决**: 裁判根据*该特定玩家*被分配的角色评估问题，并回答“是”、“否”或“不是，但也算不上错”。
5. **推理与猜测**: 在自己回合的任何时候，玩家可以选择利用该回合直接猜测自己的身份，而不是提问。
6. **结算**:
    - 猜对的玩家被标记为“已猜出”（Guessed），并不再参与后续提问回合。系统记录其最终得分（使用的提问回合越少，得分越高）。
    - 耗尽所有提问回合仍未猜对的玩家被标记为“失败”（Failed）。
    - 当所有三个玩家都成功猜出或宣告失败时，本局结束。
7. **局后复盘与学习**: 宣布本局结果。在后台，AI玩家会审查本局完整的聊天日志。它们分析成功的问题，评估错失的机会，并更新各自持久的“技能文件”，以改进其在未来对局中的推理策略。
8. **下一局或结束**: 如果还有剩余局数，则带入新角色返回到第2步。否则，统计最终总分并宣布会话获胜者。

## 5. 计分系统
- 每位玩家每局的点数取决于他们猜中身份的速度：
    - 猜对得分 = `最大提问次数 - 已使用提问次数 + 1` 分。
    - 猜测失败/耗尽次数 = 0 分。
- 在所有局数结束后，总分最高的玩家将赢得本次会话的胜利。

## 6. 核心特色
- **自主AI聊天**: 最核心的功能是观看大语言模型在结构化的社交推理游戏中进行互动。
- **单人秘密身份**: 每个玩家都有自己需要猜测的独立角色，形成并行的推理线程。
- **200名武将阵容**: 为裁判提供了一个精心挑选的包含200名三国人物的卡池，按阵营（魏、蜀、吴、群雄）、角色（主公、谋士、武将、红颜等）和时期（前期、中期、后期）进行分类。
- **本地Ollama大模型驱动**: 每个玩家、裁判和翻译器都可以由用户本地Ollama实例中不同的LLM驱动，极大地满足了测试不同模型能力的实验需求。
- **实时中文翻译**: 专用的翻译器LLM会随时将英文对话翻译成中文，并在聊天流中内联显示。
- **性格提示词 (Personality Prompts)**: 人工智能玩家可能拥有不同的“性格”或预设的提示指令（例如：激进的猜测者、谨慎的分析者）。

## 7. 用户界面 (UI)
- **视觉风格**: 经典的中国水墨画美学融合现代聊天UI元素。颜色应以深红色、金色和仿古羊皮纸白色为主。
- **布局**:
    - **顶部状态栏**: 游戏标题、当前局数状态（例如：第2局，共5局）和各自玩家的剩余回合计数器。
    - **游戏设置界面**: 初始全屏或模态框，供人类配置局数、回合数、每个角色的LLM选项以及翻译切换键。
    - **身份卡牌**: 向观众展示每个玩家所扮演秘密角色的视觉界面（对观众可见，但对对应的玩家带有隐藏标记）。
    - **主战场 (竞技场)**: 清晰可滚动的聊天对话流。不同角色的消息在视觉上要有明显区分：
        - 裁判消息（居中对齐或有显眼的头像头像）。
        - 玩家 A, B, C 的消息（彩色编码，如玩家A为蓝色，玩家B为绿色，玩家C为橙色）。
        - 翻译消息（启用翻译时，内联显示在原始消息下方，以较浅的颜色及斜体样式展示）。
    - **计分板**: 侧栏或底部区域，显示每位玩家跨越各局累计的总得分。
    - **控制面板**: 供人类观众控制AI模拟竞赛播放速度的播放/暂停/快进按钮，以及翻译功能的开/关按钮。
